# 前言

在上两篇博文中，我系统的记录了回调函数+事件这样的JS异步编程方法。实际上，处理回调的方式对解耦非常不利。于是Promise对象应运而生。

# Promise？

Promise是异步编程的一种解决方案，比传统的解决方法 —— 『回调函数+事件』更合理更强大。使用Promise对象之后可以使用一种链式调用的方式来组织代码，让代码更加简洁直观。

也就是说，有了Promise对象，就可以将异步操作以同步的操作的流程表达出来，避免了层层嵌套的回调函数。

# Callback Hell

所谓的『回调地狱』就来自回调函数的使用。而回调函数也是最基础最常用的处理js异步操作的办法。我们来看一个简单的例子：

先定义三个函数：

function fn1() {console.log('Function 1')}
function fn2() {setTimeout(() => {console.log('Function 2')}, 1000)}
function fn3() {console.log('Function 3')}

其中fn2可以看成是一个延迟了1秒执行的异步函数，现在我希望可以依次执行fn1，fn2，fn3。为了保证fn3在最后执行，我们可以把它作为fn2的回调函数：

function fn2(f) {setTimeout(() => {console.log('Funtion 2')
f()}, 1000)}

fn2(fn3)

可以看到，fn2和fn3完全耦合在一起，如果有多个类似的函数，很有可能会出现fn1(fn2(fn3(fn4(…))))这样的情况。回调地狱难以看清代码结构，让人头疼。

是的，Promise就是为了解决这些问题而来：

  1. 回调地狱，代码难以维护；
  2. Promise可以支持多个并发的请求，获取并发请求中的数据；
  3. Promise可以解决异步的问题，本身不能说Promise是异步的，它用同步的形式完成异步代码，所以写起来很简洁；

# Promise的状态

  1. 对象的状态不受外界影响

    Promise对象代表一个异步操作，有三种状态

    状态 描述
    pending 执行中，表示初始状态，既不成功也不失败
    Resolved 成功，又称Fulfille，意味着操作成功完成
    rejected 意味着操作失败。

    其中pending为初始状态，fullfilled和rejected为结束状态（结束状态表示promise的生命周期已结束）。Promise只有异步操作的结果，可以决定当前是哪一种状态，任何其他操作都无法改变这个状态。

  2. 一旦状态改变，就不会再变，任何时候都可以得到这个结果。

    Promise对象的状态改变，只有两种可能：从Pending变为Resolved和从Pending变为Rejected。

# Promise构造函数

Promise是一个构造函数，多用来实例化对象。其自身有all、reject、resolve方法，其原型上有then、catch等方法。

Promise的构造函数接收一个参数：函数类型，并且这个函数需要传入两个参数：

  • resolve ：异步操作执行成功后的回调函数
  • reject：异步操作执行失败后的回调函数

看下面栗子：

const myFirstPromise = new Promise((resolve, reject) => {
  // ?做一些异步操作，根据异步操作的成功或失败，最终会调用下面两者之一:
  //
  //   resolve(someValue); // fulfilled
  // ?或
  //   reject("failure reason"); // rejected
});

    const xhr = new XMLHttpRequest();
    xhr.open("GET", url);
    xhr.onload = () => resolve(xhr.responseText);
    xhr.onerror = () => reject(xhr.statusText);
    xhr.send();
  });
};

一般我们用new构造函数与闭包的方式来使用原生Promise。构造函数本身接收一个函数作为参数，而该函数又接收两个参数：resolve、reject，这两个参数非常重要，他们是js提供的两个函数。

Resolve函数将Promise对象从pending => resolved，即在异步操作成功时调用，并将异步操作成功返回的结果，作为参数传递出去，被then方法给接收作为参数

Reject函数将Promise对象从pending => rejected，即在异步操作失败时调用，并将异步操作返回的错误作为参数传递出去；

# 基本用法

根据上面的描述，Promise使用步骤如下：

  1. 先new一个Promise,将Promise实例化；
  2. 然后在实例化的promise可以传两个参数，一个是成功之后的resolve，一个是失败之后的reject；
  3. Promise实例生成以后，可以用then方法分别指定Resolved状态和Reject状态的回调函数；
// new构造函数方式
let p = new Promise((resolve, reject) => {
    //做一些异步操作
    setTimeout(() => {
        console.log('执行完成');
        resolve('我是成功！！');
    }, 2000);
});

// 闭包方式
var promise = function(isReady){
    return new Promise(function(resolve, reject){
        // do somthing, maybe async
        if (isReady){
          return resolve('hello world');
        } else {
          return reject('failure');
        }
    });
}
 
//Promise实例生成以后，可以用then方法分别指定Resolved状态和Reject状态的回调函数。
promise(true).then(function(value){
    // success,这里是resolve的回调函数
    console.log(value);  //hello world
}, function(err){
    // failure，这里是reject的回调函数
    console.log(err)
})

# Promise.prototype.then()

当生成了实例化Promise对象后，我们在then方法内分别指定resolve和reject状态的回调函数。因此，then的作用是：为Promise 实例添加状态改变时的回调函数。

比如这样：

let myFirstPromise = new Promise(function(resolve, reject){
    //当异步代码执行成功时，我们才会调用resolve(...), 当异步代码失败时就会调用reject(...)
    //在本例中，我们使用setTimeout(...)来模拟异步代码，实际编码时可能是XHR请求或是HTML5的一些API方法.
    setTimeout(function(){
        resolve("成功!"); //代码正常执行！
    }, 250);
});

myFirstPromise.then(function(successMessage){
    //successMessage的值是上面调用resolve(...)方法传入的值.
    //successMessage参数不一定非要是字符串类型，这里只是举个例子
    console.log("Yay! " + successMessage);
});

# 链式操作

从表面上看，Promise能够简化层层嵌套的回调写法，而实质上，Promise的精髓是“状态”，用维护状态、传递状态的方式来使得回调函数能够及时调用，它比传递callback函数要简单、灵活的多。此外，then方法返回的是一个新的Promise实例，因此可以采用链式的写法。所以使用Promise的正确场景是这样的：

function makePromise1(){
    var p = new Promise(function(resolve, reject){
        //异步操作
        setTimeout(function(){
            console.log('异步任务1');
            resolve('异步任务1传过来的值');
        }, 2000);
    });
    return p; // 返回实例化Promise对象      
}
function makePromise2(){
    var p = new Promise(function(resolve, reject){
        //异步操作
        setTimeout(function(){
            console.log('异步任务2');
            resolve('异步任务2传过来的值');
        }, 2000);
    });
    return p;            
}
function makePromise3(){
    var p = new Promise(function(resolve, reject){
        //异步操作
        setTimeout(function(){
            console.log('异步任务3');
            resolve('异步任务3传过来的值');
        }, 2000);
    });
    return p;            
}
// 链式调用
makePromise1()
.then(function(value){
    console.log(value);
    return makePromise2();
})
.then(function(value){
    console.log(value);
    return makePromise3();
})
.then(function(value){
    console.log(value);
});

异步任务1传过来的值
异步任务2
异步任务2传过来的值
异步任务3
异步任务3传过来的值

# rejected态

若Promise的状态转为rejected，我们就可以在then中传入第二个参数来执行失败时的回调：

let p = new Promise((resolve, reject) => {
        //做一些异步操作
      setTimeout(function(){
            var num = Math.ceil(Math.random()*10); //生成1-10的随机数
            if(num<=5){
                resolve(num); // resolve态
            }
            else{
                reject('数字太大了'); // reject态
            }
      }, 2000);
    });
    p.then((data) => {
            console.log('resolved',data);
        },(err) => {
            console.log('rejected',err);
        }
    );

# catch

Catch的作用是用来指定发生错误时的回调函数。它和then的第二个参数作用一样。

let p1 = function(isReady) {
	return new Promise(function(resolve, reject) {
		if(isReady) {
			return resolve('Hello World!')
		} else {
			return reject('fail!')
		}
	})
}

p1(true).then(function(v) {
  console.log('resolve！', v)
	 console.log(haha) // 未定义的变量，会被catch捕获
}).catch(function(error) {
  // 处理 p1 和 前一个回调函数运行时发生的错误
  console.log('发生错误！', error);

# all

作用：谁跑的慢，以谁为准执行回调。all接收一个Promise实例对象的数组作为参数，当这个数组内所有Promise对象的状态都变为resolved时，该方法才返回，传递给对应的回调函数。倘若promise数组中有一个被rejected，则返回第一个被reject的实例的返回值，同样传递给all后面对应的回调函数，而不管其它 promise 是否完成。

Promise的all方法提供了并行执行异步操作的能力，并且在所有异步操作执行完后才执行回调。看下面的例子：

let Promise1 = new Promise(function(resolve, reject){
	setTimeout(() => {resolve('第一个承诺！')}, 3000)
})
let Promise2 = new Promise(function(resolve, reject){
	setTimeout(() => {resolve('第二个承诺！')}, 2000)
})
let Promise3 = new Promise(function(resolve, reject){
	setTimeout(() => {resolve('第三个承诺！')}, 1000)
})

Promise.all([Promise1,Promise2,Promise3])
.then(function(result){
  // 三个都成功则成功，此时Promise1，2，3的返回值组成数组，传递给p的回调函数  
	console.log(result)
}, function(error){
  // 只要有一个失败，此时第一个被reject的实例返回值会传递给p的回调函数 
	console.log(error)
})

// output 
["第一个承诺！", "第二个承诺！", "第三个承诺！"]

# race

作用：race的意思是赛跑，谁跑的快，以谁为准执行回调。因此，promise.race也是传入一个数组，但是与promise.all不同的是，race只返回跑的快的实例的返回值，传递给对应的回调函数。

Race的使用场景：比如我们可以用race给某个异步请求设置超时时间，并且在超时后执行相应的操作，代码如下：

//请求某个图片资源
    function requestImg(){
        var p = new Promise((resolve, reject) => {
            var img = new Image();
            img.onload = function(){
                resolve(img);
            }
            img.src = '图片的路径';
        });
        return p; // 返回一个promise实例
    }
    //延时函数，用于给请求计时
    function timeout(){
        var p = new Promise((resolve, reject) => {
            setTimeout(() => {
                reject(new Error('图片请求超时'));
            }, 5000);
        });
        return p;
    }
    Promise.race([requestImg(), timeout()]).then((data) =>{
        console.log(data);
    }).catch((err) => {
        console.log(err);
    });

# Promise.resolve()

作用：将现有的对象转化为promise对象，且该promise状态为resolved。

let a = Promise.resolve(123)

let b = Promise.resolve({name:'xiaoming', age:18})

# Promise.reject()

作用：返回一个新的promise实例，此promise实例对象的状态为rejected，并且立即调用它的回调函数

const p = Promise.reject('出错了');
// 等同于
const p = new Promise((resolve, reject) => reject('出错了'))

p.then(null, function (s) {
  console.log(s)
});
// Unhandled Promise Rejection: 出错了
